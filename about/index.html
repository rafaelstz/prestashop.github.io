About

Why this blog

We decided to create this blog in order to communicate more closely with the community, especially those who contribute to the Core of PrestaShop.

This blog is about everything that has to do with PrestaShop development. It is written by the whole PrestaShop product team: developers, designers, writers, etc. Contributions are also gladly accepted!

We write about the upcoming improvement in the codebase, current and future architectures, or any behavior change that might affect you. You will also find a few How-Tos to learn how to use the new features – information that will eventually find its way in the Developer Guide or the Designer Guide.

Contribute

How about writing a guest post here? Maybe you’ve been working with PrestaShop for a while and you want to write about how you use it, or you feel a cool feature could use a highlight with sample code? Go ahead, suggest a new article — that might even inspire other people in the community to write too!

Before you start writing, please send us a short proposal at build@prestashop.com. Once we have agreed with you on the topic, you can submit your article using a simple pull request! Make sure you read the How To Write On This Blog tutorial first!

Note that all content on the Build.PrestaShop.com site is licensed under the CC0 license – meaning that it is in the Public Domain. Use it however you want!

Also, if we merge one of your cool pull request on the PrestaShop project, we will probably invite you to tell us more about it here.

Guidelines

Support

This blog is not for support. If you need help, you can search our forum or open a new thread there.

Feature requests

If you want to see some feature added to PrestaShop, please open an ‘Improvement’ ticket on the Forge.

Bug report

If you find a bug or something that doesn’t work as expected, please open a ‘Bug’ ticket on the Forge.
